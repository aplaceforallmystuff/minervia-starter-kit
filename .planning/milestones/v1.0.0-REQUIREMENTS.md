## v1 Requirements

Requirements for initial release. All mapped to phases 1-8.

### Core Installation (10/10 Complete)

- [x] **CORE-01**: Installer checks prerequisites before proceeding (Claude Code CLI, Bash version, write permissions)
- [x] **CORE-02**: Installer displays progress indication for each step (spinner or status messages)
- [x] **CORE-03**: Errors are human-readable with actionable recovery steps (not stack traces)
- [x] **CORE-04**: Installer supports --help flag showing usage, flags, and examples
- [x] **CORE-05**: Installer is idempotent (safe to re-run, skips what's already done)
- [x] **CORE-06**: Installer never deletes user content without explicit confirmation
- [x] **CORE-07**: Installer uses proper exit codes (0=success, non-zero=failure)
- [x] **CORE-08**: Installer confirms before destructive actions
- [x] **CORE-09**: Installer supports --version flag showing current version
- [x] **CORE-10**: Installer documents uninstall process (in help or docs)

### Onboarding (5/5 Complete)

- [x] **ONBD-01**: Installer presents interactive questionnaire to capture user context
- [x] **ONBD-02**: Questionnaire captures: name, vault location, role/business, key areas, working preferences
- [x] **ONBD-03**: Installer generates personalized CLAUDE.md from questionnaire answers
- [x] **ONBD-04**: Installer detects new vs existing vault and adapts behavior
- [x] **ONBD-05**: For existing vaults, installer preserves existing structure and content

### Vault Setup (4/4 Complete)

- [x] **VAULT-01**: For new vaults, installer creates PARA folder structure (00 Daily, 01 Inbox, 02 Projects, 03 Areas, 04 Resources, 05 Archive)
- [x] **VAULT-02**: Installer creates templates for daily notes, projects, and areas
- [x] **VAULT-03**: Installer creates example notes demonstrating how to use each PARA section
- [x] **VAULT-04**: Templates include proper frontmatter (YAML) for Obsidian compatibility

### Skills Installation (4/4 Complete)

- [x] **SKIL-01**: Installer copies skills to ~/.claude/skills/
- [x] **SKIL-02**: Installer copies agents to ~/.claude/agents/
- [x] **SKIL-03**: Installer records installed version in state file
- [x] **SKIL-04**: Installer records file manifest with checksums for update tracking

### Update System (6/6 Complete)

- [x] **UPDT-01**: /minervia:update command fetches latest version from git
- [x] **UPDT-02**: Update detects which files user has customized (via checksum comparison)
- [x] **UPDT-03**: Update preserves user-customized files, only updates unchanged files
- [x] **UPDT-04**: Update offers merge options for customized files (keep mine, take theirs, backup + overwrite)
- [x] **UPDT-05**: Update creates backup before modifying any files
- [x] **UPDT-06**: Update reports what changed after completion
