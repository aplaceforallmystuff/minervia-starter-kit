### E2E User Flows

#### Flow 1: Fresh Install ✓

```
User runs ./install.sh
  → parse_args processes flags
  → acquire_lock prevents concurrent runs
  → check_prerequisites validates environment
  → run_questionnaire captures user context
  → detect_vault_type identifies new vault
  → scaffold_new_vault creates PARA structure
  → install_skills copies to ~/.claude/skills/
  → install_agents copies to ~/.claude/agents/
  → install_update_script copies to ~/.minervia/bin/
  → generate CLAUDE.md from template
  → show_final_summary reports results
  → release_lock on cleanup
```

**Status:** COMPLETE

#### Flow 2: Re-run Install ✓

```
User runs ./install.sh again
  → handle_saved_answers detects previous config
  → User offered: "Use saved settings" / "Edit" / "Start fresh"
  → run_step wrapper checks is_step_complete()
  → Completed steps skipped with [SKIP] message
  → Only missing items installed
  → show_final_summary shows STEPS_INSTALLED/SKIPPED counts
```

**Status:** COMPLETE

#### Flow 3: Update System ✓

```
User runs /minervia:update
  → minervia-update.sh checks state file exists
  → fetch_latest() clones latest with git clone --depth 1
  → Compare installed vs remote version
  → create_backup() before any changes
  → scan_for_customizations() via checksum comparison
  → User prompted for customized files (keep/take/backup+overwrite)
  → apply_updates() for pristine files
  → Reports what changed
```

**Status:** COMPLETE
